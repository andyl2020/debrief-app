# Debrief v1.9.1 — Recording names and notification control

This maintenance release makes recorded files easier to identify and fixes the
active-recording notification returning after dismissal.

## What changed

- Added an editable **Recording name** field to the Recorder.
- The name can be changed before recording, while recording, while paused, or
  before retrying a failed destination save.
- Added a pencil action beside every Library recording for physical file rename.
- Recorder and Library renames preserve the real audio extension, reject blank
  names, and replace characters that Android file providers cannot safely use.
- Library renames refresh the local search index and reinstall-safe JSON sidecar.
- On Android 13+, swiping away the recording notification now hides it for the
  rest of that recording. Audio capture, screen-off operation, call handling,
  pause/resume, storage monitoring, and recovery continue normally.

## How to use

### Name a new recording

1. Open **Record**.
2. Edit **Recording name**. The fixed `.m4a` suffix shows the real output type.
3. Tap Record. You can continue editing the name during capture or while paused.
4. Tap Stop. Debrief saves the completed file using the latest valid name.

### Rename a Library file

1. Open **Library**.
2. Tap the pencil beside the recording.
3. Enter the new filename and tap **Rename**.

This physically renames the source file in the linked folder. It does not
re-encode audio or remove transcripts, comments, redactions, chapters, or AI
metadata.

### Hide the recording notification

On Android 13 or newer, swipe the active Debrief recording notification away.
It stays hidden for that recording while capture continues. Reopen Debrief to
pause, resume, or stop. The next new recording shows one new notification.

- Completely offline 48 kHz mono 128 kbps AAC/M4A Recorder with foreground

  handling, storage safeguards, rolling recovery parts, and linked-folder save.
- Linked-folder MP3/M4A/WAV/AAC Library with physical rename and batch selection.

- Optional Organize Recording and Advanced/Experimental AI Enhance tools.
- SQLCipher local database, Android Keystore API keys, and JSON sidecars.

## Limitations and edge cases

- Android 10 or newer is required. Foreground-service notifications are normally
  non-dismissible on Android 12 and older.
- Android can continue listing Debrief under **Active apps** after the visible
  notification is dismissed because the microphone foreground service remains
  active. Using that system screen's **Stop** action terminates the app and is
  different from swiping away the notification.
- Once dismissed, notification pause/resume controls are unavailable for the
  current recording; use the Recorder screen.
- Filename changes preserve the source extension. A storage provider can reject
  a collision, invalid permission, or read-only destination; Debrief keeps the
  original name and reports the failure.
- Active capture still cannot be guaranteed through force-stop, reboot/power
  loss, uninstall, permission revocation, full/failed storage, hardware failure,
  or an OEM killing the service. Finalized local parts remain recoverable.
- For critical sessions on OnePlus, enable **Allow background activity** or
  **Unrestricted** battery use if OxygenOS offers it.
- GitHub APKs are sideloaded, so Play Protect may show an unknown-developer
  warning. Install only the APK attached to this repository's release.

## Install and compatibility

Download `Debrief-v1.9.1.apk` from this release and open it on the phone. It is
production-signed to upgrade v1.9.0 in place. The universal APK includes ARM64
and is verified for 16 KB memory pages used by the OnePlus 13.
