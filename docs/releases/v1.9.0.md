# Debrief v1.9.0 — Reliable offline Recorder

This release adds completely offline, long-session audio recording to Debrief.

## What changed

- Added a dedicated **Record** bottom tab with a live timer, level meter, generated
  name, one-tap record, pause/resume, Stop, folder status, and clear save/recovery
  states.
- Added Android native `MediaRecorder` capture in a foreground microphone
  service, ongoing pause/resume notification, and screen-off wake lock.
- Added 48 kHz mono 128 kbps AAC/M4A output. Long sessions roll into protected
  local parts at roughly nine-minute boundaries, then join without re-encoding.
- Added automatic pause/resume for detected call/communication mode, warnings
  when Android temporarily silences Debrief for another microphone app, encoder
  restart attempts, and low-storage automatic pause/resume.
- Added durable interrupted-session recovery and **Retry save** /
  **Choose another folder** when the destination write fails.
- Fixed immediate transcription queueing to use the latest persisted provider and
  mobile-data setting.
- Added a private, fully gitignored local-audio test workspace and opt-in
  real-Deepgram transcription test. No fixture audio, provider key, or generated
  transcript is in this release.

## How to use the Recorder

1. Open **Record** at the bottom of Debrief.
2. If a folder is already linked, tap the large record button. Otherwise choose
   a phone folder first.
3. Grant microphone and notification access when Android asks.
4. Use Pause/Resume as needed. The ongoing notification also provides
   Pause/Resume and opens the Recorder.
5. Return to Debrief and tap Stop. Wait for the save to complete.
6. The new M4A appears in Library, ready to play or transcribe.

If **Save needs attention** appears, do not uninstall or clear app data. Tap
**Retry save**, or choose another writable folder and retry. The recoverable
local audio remains on the phone until a verified folder save succeeds.

## Current feature inventory

- Offline foreground-service Recorder with pause/resume, call and storage
  handling, local recovery, and linked-folder save.
- Linked-folder MP3/M4A/WAV/AAC Library with batch selection and automatic
  rescans.
- AssemblyAI or Deepgram transcription, provider keys stored with Android
  Keystore, configurable Original/Balanced/Data saver upload quality, speaker
  diarization, word timestamps, usage tracking, and Transcript Quality checks.
- Synchronized Media3 playback with ±1/3/5-second skip, 1× through 4× speed,
  saved position, transcript follow, transcript-only in-player search, and
  broad Library search.
- Reversible `[redacted]` transcript masking and timestamp audio muting for
  screen-recorded sharing, including per-word unredaction.
- Timestamped comments and manual set start/end/edit/delete/merge/split in the
  Chapters drawer, with colored transcript and scrubber ranges.
- Optional Organize Recording summaries/speaker suggestions/rename and optional
  Advanced AI Enhance repair diffs with raw-transcript immutability.
- Markdown export, SQLCipher-encrypted local data, and JSON sidecars for
  reinstall/rescan recovery.

## Limitations and edge cases

- Android 10 or newer is required.
- No Android app can guarantee capture through force-stop, uninstall, reboot or
  power loss, permission revocation, full/failed storage, hardware failure, or
  an OEM terminating the foreground service. Previously finalized local parts
  remain recoverable; the currently open part (up to roughly nine minutes) may
  not be after abrupt termination.
- Active audio is first stored in Debrief's app-specific area and copied to the
  linked folder at Stop. Uninstalling or clearing data removes an unsaved local
  recovery copy.
- Normal Android apps cannot record phone-call audio. Debrief pauses when Android
  reports call/communication mode. OEM/calling-app behavior can vary.
- A higher-priority app may cause Android to deliver silence to Debrief.
  Debrief keeps capture alive and warns when the platform exposes that state,
  but cannot reconstruct speech Android did not deliver.
- For critical multi-hour sessions on OnePlus or other aggressive battery
  managers, set Debrief to **Allow background activity** or **Unrestricted** if
  the OEM offers that option.
- Debrief is sideloaded from GitHub, so Play Protect may warn about an unknown
  developer. Install only the APK attached to this repository's release.
- Debug or independently signed builds cannot upgrade the production-signed app.

## Install and compatibility

Download `Debrief-v1.9.0.apk` from this release and open it on the phone.
Production-signed Debrief releases upgrade in place. The universal APK includes
ARM64 support and is checked for 16 KB native-library alignment used by devices
such as the OnePlus 13.
