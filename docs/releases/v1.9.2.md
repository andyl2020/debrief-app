# Debrief v1.9.2 — Active recording discard

This maintenance release adds a fast but deliberate way to throw away the
recording currently in progress without creating a file in the linked folder.

## What changed

- Added a trash control while the Recorder is recording or paused.
- A normal tap opens an explicit permanent-delete confirmation.
- Pressing and holding trash changes it to a delete-forever icon, gives haptic
  feedback, and discards the current session immediately.
- Discard stops microphone capture, removes every protected local part for that
  session, clears recovery state, and does not invoke normal folder export.
- A discard delivered after Stop has entered saving is ignored so deletion
  cannot race a legitimate destination write.

## How to use

### Confirmed delete

1. Start a recording, or pause one already in progress.
2. Tap the trash button to the left of Pause/Resume.
3. Review the warning and tap **Delete recording**.

Capture stops, the current session disappears, and no recording is written to
the linked folder.

### Quick hold-to-delete

Press and hold the trash button. The icon changes to delete-forever and the phone
gives haptic feedback when the long press is accepted. The current session is
discarded immediately without a second dialog.

Use **Stop** instead whenever the audio should be kept. Stop joins protected
parts and saves one normal M4A file; trash deletes the current session.

## Limitations and edge cases

- Discard is permanent. A normal tap has a confirmation dialog, but a completed
  press-and-hold intentionally deletes immediately.
- Only the current app-private recording session is removed. Previously saved
  Library files are not affected.
- Trash is available while recording or paused. Once Stop has begun final
  saving, Debrief completes the save and ignores any stale discard action.
- If a folder save has already failed, Debrief keeps its existing
  **Save needs attention** recovery flow; this active-session trash control is
  not a replacement for recovery management.
- Active capture still cannot be guaranteed through force-stop, reboot/power
  loss, uninstall, permission revocation, full/failed storage, hardware failure,
  or an OEM killing the service. Finalized local parts remain recoverable unless
  the user deliberately discards the session.
- Android 10 or newer is required. For critical sessions on OnePlus, enable
  **Allow background activity** or **Unrestricted** battery use if OxygenOS
  offers it.
- GitHub APKs are sideloaded, so Play Protect may show an unknown-developer
  warning. Install only the APK attached to this repository's release.

## Install and compatibility

Download `Debrief-v1.9.2.apk` from this release and open it on the phone. It is
production-signed to upgrade v1.9.1 in place without removing app data. The
universal APK includes ARM64 and is verified for 16 KB memory pages used by the
OnePlus 13.
